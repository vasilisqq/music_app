 Нужно довести режим «▶️  Играть по урокам» до полноценного UX:
 - Кнопка «Назад» на экране списка уроков должна возвращать к списку тем (а не на главную).
 - Список уроков внутри темы должен быть «красивым» вертикальным списком (не таблица).
 - При выборе урока должен открываться экран урока: показ названия/описания + сцена как в creator, но без редактирования нот.
 - На экране урока должны быть 2 действия: «Слушать» и «Повторить».
   - «Слушать»: проигрывание мелодии со звуком нот.
   - «Повторить»: проигрывание без звука нот, но со звуком метронома, при этом включено отслеживание кликов.
   - По завершению «Повторить» показываем окошко с точностью (accuracy).
   - Если accuracy ≥ 80% — отмечаем урок completed (POST /progress/lesson/{id}/complete), чтобы разблокировался следующий.
 - Ввод для отслеживания кликов: Space (как сейчас).

 Проверка вручную (end-to-end)

 1. Запустить сервер и применить миграции: alembic upgrade head, старт FastAPI.
 2. Запустить клиент: uv run app/main.py.
 3. Главная → «Играть по урокам»:
   - Открывается список тем.
 4. Выбрать тему → показывается красивый вертикальный список уроков.
 5. Back на экране уроков → возвращает к темам (не на home).
 6. Открыть урок:
   - Видно название и описание.
   - На сцене ноты отображаются, клик по линиям/пробелы не добавляют новых нот.
 7. Нажать «Слушать» → мелодия проигрывается со звуком нот.
 8. Нажать «Повторить» → нет звука нот, но есть клик метронома; Space засчитывается; после завершения показывается точность.
 9. Если точность ≥ 80% → урок отмечается completed, и следующий урок в списке становится доступным.
