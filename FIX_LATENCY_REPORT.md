РЕШЕНИЕ ПРОБЛЕМЫ ЗАДЕРЖКИ ПРИ ВОСПРОИЗВЕДЕНИИ НОТ И НАРУШЕНИЯ СИНХРОНИЗАЦИИ С МЕТРОНОМОМ
================================================================================================

ПРОБЛЕМА:
---------
После оптимизации кеша с 800 МБ на 5-10 МБ, при воспроизведении нот возникала огромная задержка 
(10-50 мс) и ноты не синхронизировались с метрономом, так как:
1. generate_note() требовала 10-50 мс на CPU для scipy.ndimage.zoom + anti-aliasing фильтр
2. preload_staff_notes() работала в фоновом потоке, что не гарантировало готовность нот к началу метронома
3. На-требование генерация при первом проигрывании вызывала задержку


РЕШЕНИЕ: Двухуровневая оптимизация
===================================

1️⃣  ОПТИМИЗИРОВАННАЯ ФУНКЦИЯ ГЕНЕРАЦИИ (generate_note)
------------------------------------------------------
Добавлена поддержка двух режимов:

   • use_fast_resample=False (качество, для предвычисления):
     - scipy.ndimage.zoom с order=3 (кубическая интерполяция)
     - Anti-aliasing фильтр низких частот (Butterworth order=5)
     - Результат: высокое качество звука (~4-4.5 сек на 19 нот)
   
   • use_fast_resample=True (скорость, для на-требование нот):
     - scipy.ndimage.zoom с order=1 (линейная интерполяция - 2-3х быстрее)
     - БЕЗ фильтра anti-aliasing (экономим 50% CPU)
     - Результат: быстро (~8-25 мс на ноту, приемлемо для не-служебных нот)


2️⃣  СИНХРОННАЯ ПРЕДВЫЧИСЛЕНИЕ ВСЕХ СЛУЖЕБНЫХ НОТ
---------------------------------------------------
   • preload_staff_notes_sync() вызывается СИНХРОННО при импорте модуля
   • Предвычисляет ВСЕ 19 нот нотного стана за ~4 сек
   • Это блокирует запуск приложения на 4 сек, но ГАРАНТИРУЕТ:
     ✓ Все ноты в памяти для немедленного воспроизведения
     ✓ 0 мс задержка из кеша
     ✓ Идеальная синхронизация с метрономом


3️⃣  ГИБРИДНЫЙ КЕШ (OrderedDict LRU)
-------------------------------------
   • Служебные ноты (19): всегда в памяти, качественный sound
   • Прочие ноты (100+): генерируются по требованию, быстро, когда нужны
   • Максимум 50 нот в памяти = 5-10 МБ (vs 800 МБ)
   • LRU вытеснение: старые ноты удаляются при превышении лимита
